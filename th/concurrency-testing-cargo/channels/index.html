Channels

การส่งข้อความด้วย std::sync::mpsc

หัวข้อที่มีชื่อว่า “การส่งข้อความด้วย std::sync::mpsc”

standard library ของ Rust มี channel แบบ multi-producer, single-consumer (mpsc) สำหรับส่งค่าระหว่าง thread channel มีสองปลาย:

  • tx — transmitter (ผู้ส่ง) คุณสามารถ clone มันเพื่อสร้าง producer หลายตัว
  • rx — receiver (ผู้รับ) มี receiver เพียงตัวเดียวต่อ channel

mpsc::channel() ส่งคืนคู่ (Sender<T>, Receiver<T>) ประเภท T ถูก infer จาก send call แรก

การส่งค่าข้าม Thread

หัวข้อที่มีชื่อว่า “การส่งค่าข้าม Thread”

transmitter เป็น Send ดังนั้นคุณสามารถย้ายมันเข้าไปใน thread closure เมื่อ tx.send(value) ถูกเรียก ค่านั้นจะถูก ย้าย เข้าไปใน channel — thread ผู้ส่งไม่ได้เป็นเจ้าของมันอีกต่อไป

Multiple Producers

หัวข้อที่มีชื่อว่า “Multiple Producers”

Clone tx เพื่อสร้าง sender หลายตัว แต่ละ clone มี handle ของตัวเองไปยัง channel เดียวกัน receiver เห็นข้อความตามลำดับที่มาถึง (non-deterministic ข้าม thread) ดังนั้นให้รวบรวมและ sort ก่อนพิมพ์

rx.iter() ให้ค่าจนกว่า Sender handle ทั้งหมดจะถูก drop การ drop tx ตัวเดิม (หลัง clone) จำเป็นเพื่อให้ iterator รู้ว่าจะหยุดเมื่อใด

Fan-Out: การกระจายงาน

หัวข้อที่มีชื่อว่า “Fan-Out: การกระจายงาน”

producer คนเดียวสามารถส่ง work item ไปยัง pool ของ thread แต่ละ worker รับ item ผ่าน channel ของตัวเอง

ตัวอย่าง Playground แบบสมบูรณ์

หัวข้อที่มีชื่อว่า “ตัวอย่าง Playground แบบสมบูรณ์”
Open in Rust Playground
use std::thread;
use std::sync::mpsc;

fn main() {
    let (tx, rx) = mpsc::channel();
    let mut handles = Vec::new();

    for i in 0..5 {
        let tx = tx.clone();
        let handle = thread::spawn(move || {
            let msg = format!("result from thread {}", i);
            tx.send(msg).unwrap();
        });
        handles.push(handle);
    }
    drop(tx);

    for h in handles {
        h.join().unwrap();
    }

    let mut msgs: Vec<String> = rx.iter().collect();
    msgs.sort();
    for m in &msgs {
        println!("{}", m);
    }
}

🦀 Channels ย้าย Ownership ระหว่าง Thread

เมื่อคุณเรียก tx.send(value) ownership ของ value จะถูกย้ายเข้าไปใน channel thread ผู้ส่งไม่สามารถเข้าถึงมันได้อีกต่อไป thread ผู้รับจะเป็นเจ้าของค่านั้นหลัง rx.recv() การย้าย ownership นี้คือเหตุผลที่ channel ปลอดจาก data race โดยโครงสร้าง — มีเพียง thread เดียวที่เป็นเจ้าของค่าในแต่ละจุดของเวลา

mpsc ใน std::sync::mpsc ย่อมาจากอะไร?
อะไรเกิดขึ้นกับค่าหลังจาก tx.send(value) ถูกเรียก?
ทำไมต้อง drop tx ตัวเดิม (drop(tx)) ก่อนรวบรวมจาก rx.iter()?
คุณสร้าง producer หลายตัวสำหรับ receiver ตัวเดียวได้อย่างไร?
